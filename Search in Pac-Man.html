Search in Pac-Man


You will learn to solve problem by searching.

Search in Pac-Man is a Python package that allows you to practice various well-known uninformed and informed searching algorithms in different scenarios.

Things to learn in Pac-Man (a)

  • Python basics

  • Breadth-first search (BFS)

  • Depth-first search (DFS)

  • The data structures "queue" and "stack"

Things to learn in Pac-Man (b)

  • Define efficient search state representation
    An efficient representation can greatly reduce search space.

  • A* search algorithm
    Probably your first implementation of informed search algorithm.

  • The data structure "priority queue"


Prerequisites

  • Python 2 (2.7 is recommended), available at https://www.python.org/downloads/
    Python 3 does not work for the Pac-Man program.

  • The "search.zip"

Submission

  • For Pac-Man (a), please upload ONLY the search.py (no zipping) to eeclass.

  • For Pac-Man (b), please upload ONLY the search.py and searchAgents.py (no zipping) to eLearn.

  • All unexpected files will be ignore when testing and scoring.

  • Remove/comment out your debugging code snippets before submitting.

Getting started on Windows10

The following steps will run a test program of Pac-Man.

  • Unzip search.py and locate the unzipped content in File Manager

  • Press "shift" + "right click" on the blank in File Manager

  • Click "在這裡開啟 PowerShell 視窗" in the pop-up menu

  • Try a testing command in the PowerShell window:

    • If you have only Python 2 installed on your computer, enter
      python pacman.py -l tinyMaze -p SearchAgent -a fn=tinyMazeSearch

    • If you have both Python 2 & Python 3 installed on your computer, enter
      py -2 pacman.py -l tinyMaze -p SearchAgent -a fn=tinyMazeSearch

You should see a game window with a Pac-Man running automatically to the goal of maze. If you don't please check again whether you have skipped any of the above steps.

Getting started on MacOS/Linux

The following steps will run a test program of Pac-Man.

  • Unzip search.py

  • Open a terminal and change the working directory (use the cd command) to the path where you store the unzipped files

  • Try a testing command in terminal:

    • If you have only Python 2 installed on your computer, enter
      python pacman.py -l tinyMaze -p SearchAgent -a fn=tinyMazeSearch

    • If you have both Python 2 & Python 3 installed on your computer, enter
      python2 pacman.py -l tinyMaze -p SearchAgent -a fn=tinyMazeSearch

You should see a game window with a Pac-Man running automatically to the goal of maze. If you don't please check again whether you have skipped any of the above steps.

About the assignments

Must carefully read the search.html file

  • Details for Pac-Man (a) please see the section "Finding a Fixed Food Dot using Search Algorithms" (Question 1 & 2)

  • Details for Pac-Man (b) please see the section "A* search" (Question 4) and "Finding All the Corners" (Question 5 & 6)

To start with Pac-Man (a)

  • Implement the functions depthFirstSearch() and depthFirstSearch() in search.py

  • Read the comments before you start.

  • Remove util.raiseNotDefined() before testing your code

  • Do NOT add/edit/delete anything outside of the above-mentioned functions

To start with Pac-Man (b)

  • Implement the function aStarSearch() in search.py

  • Find the class CornersProblem in searchAgents.py and implement

    • the function getStartState(self)

    • the function isGoalState(self, state)

    • the function getSuccessors(self, state)

  • Implement the function cornersHeuristic(state, problem) in searchAgents.py

  • Read the comments before you start.

  • Remove util.raiseNotDefined() before testing your code

  • Do NOT add/edit/delete anything outside of the above-mentioned functions

Useful notes

  • The data structure queue, stack, and priority queue along with their own push() and pop() methods are already implemented in utils.py

    • You can store all types of variable in them, such as integer, string, list, tuple, dictionary, ...

    • Example:
      candidates = util.Queue()
      candidates.push("apple")
      print(candidate.pop())

  • The file commands.txt contains some testing commands you can play with.

  • To speed up your testing, append --frameTime 0 to the command you are testing.

  • To downsize a game window, append -z .5 to the command you are testing.
